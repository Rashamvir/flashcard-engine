# Flashcard Engine

A web app that turns any PDF into AI-generated flashcards with spaced repetition. Upload a PDF, get a smart deck of cards generated by Llama 3, and practice with a scheduling system that adapts to your memory.

Built as part of the Cuemath AI Builder Challenge.

## What It Does

- **Upload any PDF** — lecture notes, textbook chapters, study material
- **AI generates 15–20 deep flashcards** — not shallow definitions, but cards covering concepts, relationships, edge cases, and worked examples
- **Practice with a 3D flip card interface** — click to reveal the answer
- **SM-2 spaced repetition** — cards you know well fade into the future, cards you struggle with keep appearing
- **Track your mastery** — see mastered vs shaky cards, study streak, and per-deck progress charts
- **Manage multiple decks** — browse, search, practice again, or delete

## How It Works

### 1. PDF Ingestion
The uploaded PDF is parsed using `pypdf`. Text is extracted and split into chunks of ~3000 characters to stay within token limits.

### 2. Card Generation
Each chunk is sent to Groq's Llama 3.3-70b model with a carefully engineered prompt that instructs the model to generate cards like a great teacher — covering key concepts, definitions, relationships, edge cases, and worked examples. The model returns structured JSON which is parsed and saved to SQLite.

### 3. Spaced Repetition (SM-2)
After every practice session, each card's schedule is updated using the SM-2 algorithm:

```
If quality >= 3 (Almost or Got it):
    interval = interval × ease_factor
    ease_factor = max(1.3, ease_factor + 0.1 - (5 - quality) × 0.08)
    repetitions += 1
Else (Missed):
    interval = 1 day
    repetitions = 0

next_review = today + interval
```

Cards you know well get pushed further into the future. Cards you miss reset to tomorrow.

### 4. Practice Session
Cards due today are served one at a time. The interface uses a CSS 3D flip card — question on the front, answer on the back. After flipping, you rate yourself: **Got it**, **Almost**, or **Missed**. The next card loads without a page reload via JavaScript fetch calls.

### 5. Progress Tracking
The Progress page shows per-deck stats (total, mastered, learning, due today), a study streak counter based on session history, and a Chart.js bar chart showing mastery breakdown across decks.

## Running Locally

**Prerequisites:** Python 3.10+, a Groq API key (free at [console.groq.com](https://console.groq.com))

```bash
# Clone the repo
git clone https://github.com/rashamvirkaur/flashcard-engine.git
cd flashcard-engine

# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Add your Groq API key
echo "GROQ_API_KEY=your_key_here" > .env

# Run the app
python app.py
```

Open [http://localhost:5000](http://localhost:5000)

## Key Design Decisions

**Why Flask over Django or FastAPI?**
Flask is lightweight and keeps the codebase simple and readable. For a single-user app with straightforward routes, it's the right tool.

**Why SQLite?**
No need for a hosted database at this scale. SQLite is fast, zero-config, and sufficient for storing decks, cards, and session history.

**Why Groq over OpenAI?**
Groq's free tier is genuinely free (no credit card required), extremely fast (sub-3 second generations), and Llama 3.3-70b produces high-quality structured output when prompted correctly.

**Why SM-2 over a simpler algorithm?**
SM-2 is the algorithm behind Anki — the most widely used flashcard app in the world. It's well-studied, explainable, and produces noticeably better learning outcomes than simple interval doubling.
